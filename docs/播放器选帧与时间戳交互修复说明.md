# 播放器选帧与时间戳交互修复说明

## 目标

修复全屏播放器、选帧标记和时间戳导出中的交互问题。不要重做现有视觉架构，不要改变普通播放器、移动端时间线和知识笔记生成逻辑。

涉及文件：

- `frontend/src/components/ui/media-expanded.tsx`
- `frontend/src/components/ui/audio-player.tsx`
- `frontend/src/components/TimelinePanel.tsx`
- `frontend/src/pages/Workstation.tsx`

## 任务一：让“复制时间戳”更容易发现

### 当前问题

“复制时间戳”藏在时间线标题右侧的汉堡菜单中。用户需要先理解这个菜单，再点开并寻找功能，入口过深。

### 修改要求

1. 时间线面板打开后，在标题栏右侧直接显示复制图标按钮。
2. 按钮使用 `Copy` 图标，桌面宽度足够时显示“复制时间戳”，窄屏只显示图标。
3. 保留 tooltip：`复制时间戳`。
4. 点击按钮立即执行复制，不再弹出二级菜单。
5. 没有生成知识笔记、正在导出或没有时间线内容时禁用。
6. 成功提示为“时间戳已复制”，失败提示为“复制失败”。
7. 删除仅为这一项存在的 `DropdownMenu` 和汉堡按钮，避免留下空菜单。

建议标题栏结构：

```text
[时间线]                         [52 段] [复制时间戳]
```

## 任务二：补齐全屏播放器控制

### 当前问题

`media-expanded.tsx` 使用：

```tsx
controls={!pickMode}
```

进入选帧模式后，浏览器原生控制栏被关闭，但组件没有补齐播放/暂停、音量、退出全屏等功能。因此选帧时只能拖动简化进度条，无法完成正常播放操作。

### 修改要求

全屏状态始终使用同一套自定义控制栏，不要依赖浏览器原生 `controls`。至少包含：

1. 播放/暂停按钮。
2. 静音/取消静音按钮。
3. 水平音量 Slider。
4. 当前时间和总时长。
5. 可点击、可拖动的完整进度条。
6. “退出全屏”按钮，调用现有 `handleClose()`，关闭后将时间同步回普通播放器。
7. 选帧开关和帮助按钮。

新增并维护以下状态：

```ts
const [isPlaying, setIsPlaying] = useState(false);
const [volume, setVolume] = useState(1);
```

视频事件必须同步状态：

```tsx
onPlay={() => setIsPlaying(true)}
onPause={() => setIsPlaying(false)}
onVolumeChange={syncVolume}
```

退出全屏时必须保留：

- 当前播放时间；
- 音量与静音状态；
- 打开全屏前是否正在播放。

不要让全屏播放器和普通播放器同时播放。

## 任务三：让全屏进度条支持点击跳转

### 当前问题

全屏进度条只是一个显示进度的 `div`，不能点击或拖动。黄色标记虽然使用 `timestamp / duration`，但轨道非常短，视觉上全部堆在左侧，用户会认为比例错误。

### 修改要求

1. 进度条独占一整行，宽度为控制面板可用宽度的 100%。
2. 最小可点击高度 24px，真实轨道可保持 4-6px。
3. 点击轨道时，根据鼠标位置计算目标时间：

```ts
const rect = event.currentTarget.getBoundingClientRect();
const ratio = Math.min(1, Math.max(0, (event.clientX - rect.left) / rect.width));
media.currentTime = ratio * duration;
```

4. 支持拖动播放头；优先复用普通播放器已有的 `CustomSlider`，不要另写一套不一致的 Slider。
5. 黄色标记点必须定位在同一个轨道容器内：

```ts
left: `${(timestamp / duration) * 100}%`
```

6. 标记点点击后立即跳到对应时间，并阻止事件冒泡。
7. 标记点 hover/focus 时显示时间，如 `01:13`。
8. 相邻时间过近时可以视觉重叠，但数据不能被擅自合并或改变。

### 验收样例

总时长 `04:52`（292 秒）时：

- `00:10` 应位于约 `3.4%`；
- `00:30` 应位于约 `10.3%`；
- `01:13` 应位于约 `25%`；
- 点击轨道中点应跳到约 `02:26`。

## 任务四：固定“标记当前帧”，避免越点越跑

### 当前问题

已选时间使用 `flex-wrap` 与“标记当前帧”共处一行。时间点增加后发生换行，按钮位置不断变化，用户无法在同一位置连续点击。

### 推荐布局

把选帧区拆成固定操作区和可滚动结果区：

```text
[标记当前帧] [已选 30 帧]                 [清空] [处理全部(30)]
[00:10] [00:11] [00:12] ...  横向滚动，不换行
```

### 修改要求

1. “标记当前帧”始终固定在左侧，不随标记数量移动。
2. “清空”和“处理全部”始终固定在右侧。
3. 中间的时间标签使用单行横向滚动：

```css
display: flex;
flex-wrap: nowrap;
overflow-x: auto;
min-width: 0;
```

4. 不允许时间标签把控制面板撑高到遮挡视频。
5. 点击时间标签本体跳到该时间；删除只由标签内垃圾桶按钮执行。
6. 新增标记后自动把标签滚动到最右侧，但不能移动“标记当前帧”按钮。
7. 重复标记继续沿用现有去重提示。
8. 建议显示计数“已选 N 帧”，减少用户逐个数标签的压力。

移动端可分成三行，但操作按钮位置仍需稳定：

```text
[标记当前帧] [已选 N 帧]
[横向滚动的时间标签]
[清空] [处理全部(N)]
```

## 任务五：状态同步与边界情况

必须验证以下情况：

1. 普通播放器播放到某处后打开全屏，时间不跳回 0。
2. 全屏内跳转、暂停或调整音量，关闭后普通播放器保持相同状态。
3. 进入/退出选帧模式不会重置播放时间。
4. `duration` 为 `0`、`NaN` 或尚未加载时，不计算百分比、不允许跳转。
5. 连续快速点击“标记当前帧”不会插入重复时间。
6. 处理期间禁用“标记”“删除”“清空”和“处理全部”，避免提交过程中列表变化。
7. 按 `Escape`、点关闭按钮、点遮罩关闭时都走同一个 `handleClose()`。
8. 图片和纯音频预览不能因为视频控制栏改动而回归。

## 响应式要求

至少检查以下视口：

- `1920 × 1080`
- `1366 × 768`
- `768 × 1024`
- `390 × 844`

要求：

- 控件不互相遮挡；
- 文本不溢出按钮；
- 全屏视频始终保留完整画面比例；
- 控制面板最多占视口高度约 30%，标签区只横向滚动；
- 移动端保留不小于 44px 的主要点击区域。

## 不要修改

- 不要恢复自动抽帧 OCR。
- 不要改变批量选帧 API。
- 不要改变证据块数据结构。
- 不要改变知识笔记提示词。
- 不要重新设计主页面架构或颜色。
- 不要提交 `Smart Scribe.lnk`、数据库、storage 或构建缓存。

## 验收流程

1. 打开一个 4-5 分钟视频。
2. 普通播放器播放到 `00:30`，打开全屏，确认仍为 `00:30`。
3. 在全屏内点击进度条约 25% 处，应跳到约 `01:13`。
4. 调整音量并暂停，再退出全屏，普通播放器状态保持一致。
5. 连续标记至少 30 帧，鼠标不移动也能持续点击“标记当前帧”。
6. 点击任一时间标签，视频跳到对应时间。
7. 检查黄色标记与总时长比例一致。
8. 打开时间线，直接点击“复制时间戳”，不经过二级菜单。
9. 分别在桌面、平板和手机视口截图检查。
10. 运行 `npm run build`，确认构建通过。
